2. .java 파일이 실행되는 과정을 설명하시오.

자바 코드 -> 클래스 로더 -> 바이트 코드(.class) -> JVM -> 바이너리 코드 -> OS
의 순서로 실행된다. 자바 코드를 클래스 로더가 바이트 코드로 컴파일 하고, 바이트 코드를 JVM이 다시 바이너리 코드로 컴파일 하여 OS와 상호작용하여 실행되는 구조이다.

4. 콘텍스트 스위칭이 일어나는 상황과 과정을 설명하시오. (Process p1과 Process p2가 있다고 가정)

P1이 실행중이라고 가정할 때, 입출력 이벤트 등으로 인해 인터럽트가 발생하면 p1의 실행이 중단되고 대기 상태로 전환된다. 이후 P2로 cpu를 점유하는 프로세스가 변경되는데, 이 과정을 콘텍스트 스위칭이라고 한다. p2로 프로세스가 변경되면서 cpu는 p2의 PCB를 읽어 정보를 확인하고 연산을 이어간다. p1의 인터럽트가 해결되면 다시 p1의 PCB를 읽어 연산을 이어간다.

7. 빈칸에 알맞은 말을 넣으시오
    중기 스케줄링    : 스왑 아웃, 스왑 인을 통해 메모리에 로드되는 프로세스 수를 동적으로 조절한다.
    장기 스케줄링   : 준비 큐에 어떤 프로세스를 넣을지 결정해 메모리에 올라가는 프로세스 수를 조절한다.
    단기 스케줄링    : 준비큐에 있는 대기 프로세스 중 어떤 프로세를 다음으로 실행할지 결정한다.

8. 비선점형 스케줄링은 무엇이고 방식 2가지와 선점형 스케줄링은 무엇이고 방식 3가지를 적으시오.

비선점형 스케줄링은 한 프로세스가 실행중일 때 프로세스 종료 전까지 다른 프로세스가 cpu를 점유할 수 없는 것을 말한다. 방식은 FCFS 스케줄링과 SJF 스케줄링이 있다.
선점형 스케줄링은 실행중인 프로세스를 종료하고 다른 프로세스로 교체가 가능한 것을 말한다.
방식에는 SRTF 스케줄링과 RR 스케줄링, 멀티 레벨 스케줄링이 있다.

9. 내부,외부 단편화란?
외부 단편화 : 고정 할당 방식으로 메모리를 할당할 경우, 가용 메모리 공간이 8M 2M 라 가정할 때, 10M 프로세스 메모리 영역이 가용 메모리 공간보다 커서 할당이 불가능한 상황을 말한다. 
내부 단편화 : 프로세스 메모리 영역이 가용 메모리 공간보다 작아서 내부 메모리가 남는 현상을 말한다.

10. 최초 적합,최악 적합이란?

최초 적합 : 프로세스 메모리 크기에 맞는 메모리 공간에 프로세스를 로드하는 것을 말한다.
최악 적합 : 프로세스 메모리 크기 이상인 가용 메모리 공간에서 가장 큰 영역에 프로세스를 로드하는 것을 말한다. 전체 가용 메모리 공간을 탐색해야 한다.

11. 비연속 메모리 할당 방법 2가지

페이징 : 프로세스 메모리 와 물리 메모리를 페이지와 프레임으로 나누어 메모리를 분할하는 방식을 말한다. 페이지 테이블을 사용하며 페이지 정보와 프레임 주소 값이 테이블에 저장된다. 페이지 테이블은 PCB에 저장된다.
세그멘테이션 : 메모리를 세그먼트라는 논리적 단위로 나누어 분할한다. 세그먼트 테이블을 사용한다.

12. 시간,공간적 지역성이란?

시간 지역성 : 최근 데이터를 사용할 가능성이 높은 것을 말한다.
공간적 지역성 : 사용 데이터 근처 데이터를 사용할 가능성이 높은 것을 말한다.

13. 요구페이징이 어떻게 동작하는지 설명하시오.

필요한 페이지만 페이징 테이블로 처리하고 필요하지 않은 페이지는 디스크에 저장하는 방식을 말한다. 페이지 테이블에 필요한 프레임이 있을 때 v 없을 때 i를 반환하는데, i가 반환되는 상황을 페이지 폴트라고 한다. 페이지 폴트가 발생하면 OS는 프레임 주소 값이 유효하지 않은지, 해당 프레임 주소 값이 로드되지 않은 영역인지 확인하고 로드되지 않은 영역이라면 디스크에서 해당 부분을 빈 프레임에 로드한다. 이후 v가 반환되어 다시 정상 동작한다.
