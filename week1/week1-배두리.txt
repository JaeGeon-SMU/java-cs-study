자바 스터디 1주차 문제 - 배두리

1. jvm이란 무엇이고 사용하는 이유는 무엇인가?
자바 버추얼 머신의 약자로 자바 프로그램을 실행하기 위해 반드시 필요한 가상 머신이다.
JVM을 사용하면 자바 프로그램을 한 번만 프로그래밍하면 OS에 독립적으로 사용이 가능하다는 이점이 있어서 사용된다.  

2. jit 컴파일러가 무엇이고 어떻게 동작하는가?

JVM은 인터프리터 방식으로 컴파일되는데, 이 때 같은 메소드도 변경되면 여러번 호출된다는 단점이 있다. 따라서 속도가 느려지는 문제가 발생하는데, jit 컴파일러는 이런 문제를 보완하는 컴파일러다. 
코드를 바이트 코드로 컴파일한 뒤, 변경되는 부분이 있을 경우 네이티브 코드로 바꾸어 저장하는 방식으로 동작한다.

3. .java 파일이 실행되는 과정을 설명하시오.

프로그래머가 코드를 작성하여 main.java 파일이 존재한다고 가정할 때, 1차적으로 java 파일이 main.class 파일로 jvm이 이해할 수 있는 바이트 코드로 컴파일 된다.  이후 jvm을 통해  바이트 코드가 바이너리 코드로 컴파일 되어 OS와 상호작용 한다. 

4. 변수의 종류에는 인스턴스 변수,클래스 변수, 지역 변수가 있다. 각 변수의 스코프와 라이프 타임에 대해 설명하시오.

지역변수는 선언된 블록을 스코프로 가진다.

5. 타입 프로모션과 캐스팅에 대해 설명하시오.
캐스팅은 넓은 범위 타입에서 좁은 범위 타입으로 변환할 때 사용한다. 
(변환할 타입)데이터 의 형태로 소괄호 안에 타입을 기재하여 변경한다.

타입 프로모션은 좁은 범위 타입에서 넓은 범위 타입으로 변환할 때 사용한다.
데이터 손실이 없기 때문에 자동 형변환이 가능하다. 아래는 예시다..
int a = 1;
long b = a;


6. 프리미티브 타입과 레퍼런스 타입에 대해 간단히 설명하시오.
프리미티브 타입은 기본값을 가지며, null이 허용되지 않는 타입이다. 값이 직접 저장된다.
8가지로 boolean, char, byte, short, int, long, float, double이 있다.

레퍼런스 타입은 프리미티브 타입을 제외한 나머지 타입을 말한다. 사용자가 지정한 클래스나 열거형 타입이 이에 해당한다. 값이 직접 저장되지 않고 주소를 저장하여 그 주소를 참조하는 방식으로 동작한다. null이 허용된다.

7. 시스템 콜이란?

운영체제는 중요 자원에 사용자가 쉽게 접근하는 것을 막기 위해 사용자 모드와 커널 모드로 구분하여 동작한다. 프로세스가 자원을 요청하기 위해서는 운영체제를 통해 커널 모드를 요청해야 하는데, 이때 호출하는 함수가 시스템 콜이다.

프로세스가 시스템 콜을 호출하여 운영체제에게 작업을 요청하면 운영체제는 해당 작업을 수행한 뒤 다시 시스템 콜을 통해 프로세스에 응답한다.
